# Docker Compose 인프라 구조 및 설정 가이드

본 문서는 `spring_boot_reservation` 프로젝트의 로컬 개발 및 테스트 환경을 위해 구성된 `docker-compose.yml` 파일의 세부 설정과 강화된 개인화/보안 설정에 대해 설명합니다.

## 1. 개요

프로젝트 운영에 필요한 4가지 핵심 인프라(MariaDB, Redis, RabbitMQ, Kafka)를 컨테이너 기반으로 한 번에 실행하고 관리할 수 있도록 구성했습니다.

## 2. 포함된 서비스 목록

- **MariaDB** (Port: `3306`): 메인 관계형 데이터베이스 (RDBMS)
- **Redis** (Port: `6379`): 인메모리 데이터 저장소, 캐싱 및 세션 관리
- **RabbitMQ** (Port: `5672` / `15672(Web UI)`): 메시지 브로커 (앱 통신 및 라우팅)
- **Kafka & Zookeeper** (Port: `9092` / `2181`): 대용량/실시간 이벤트 스트리밍 시스템

---

## 3. 주요 개인화 및 보안 강화 설정

초기 뼈대 구조에서 보안, 데이터 안정성, 향후 확장을 고려하여 다음과 같이 설정을 고도화했습니다.

### 3.1. 환경 변수(`.env`) 분리 및 은닉

- **이유**: `yml` 파일에 DB 패스워드나 관리자 계정이 평문으로 작성되면 Git 등에 업로드될 때 심각한 보안 이슈가 발생합니다.
- **반영 사항**: 모든 계정 정보와 패스워드를 `.env` 파일로 분리하고, `.gitignore`에 자동 추가하여 소스 코드에서 민감한 정보가 노출되지 않도록 조치했습니다.
  - 예: `MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}`

### 3.2. 데이터 영속성 보장 (전체 Volumes 매핑)

- **이유**: 컨테이너가 내려가거나 삭제되었을 때 내부 데이터가 함께 소멸하는 문제를 방지합니다. 기존에는 DB에만 설정되어 있었습니다.
- **반영 사항**: Redis(캐시), RabbitMQ(메시지 큐), Kafka/Zookeeper(이벤트 스트림) 전체 서비스에 `volumes`를 매핑하여 재시작 시에도 데이터가 유지되도록 안전장치를 마련했습니다.

### 3.3. 타임존(Timezone) 한국 시간 동기화

- **이유**: 기본 Docker 컨테이너는 `UTC(협정 세계시)`를 따르므로 로그와 DB 기록이 한국 시간(KST)과 9시간 차이가 납니다.
- **반영 사항**: 각 컨테이너 인스턴스에 `TZ: ${TZ}` 환경 변수(기본값: `Asia/Seoul`)를 주입하여 한국 시간으로 로그와 시스템 타임이 남을 수 있도록 일치시켰습니다.

### 3.4. Kafka 내부/외부 네트워크 리스너 분리

- **이유**: Kafka는 호스트 서버 외에도 동일 도커 네트워크 내부의 서비스 간 통신 시 접속하는 동적 IP/Port 경로가 구별되어야 합니다.
- **반영 사항**: `KAFKA_LISTENERS` 및 `KAFKA_ADVERTISED_LISTENERS`를 분리했습니다.
  - 외부 (Spring Boot 앱이 호스트에서 접속할 때): `localhost:9092`
  - 내부 (다른 컨테이너들이 접속할 때): `kafka:29092`

### 3.5. 사용자 정의 브리지 네트워크 구성

- **이유**: 도커 기본 네트워크에 의존하기보다, 프로젝트 전용 네트워크 공간을 확립하는 것이 서비스 간 이름(DNS) 기반 통신 관리에 이점이 큽니다.
- **반영 사항**: `reservation_network`라는 전용 네트워크 구역에 모든 서비스를 바인딩 시켰습니다. 추후 Spring Boot 앱 역시 도커로 올릴 때 이 네트워크에 추가하면, 로컬호스트를 거치지 않고 직접 내부 통신이 가능해집니다.

### 3.6. Redis 패스워드 보안 추가

- **이유**: 기본 Redis 이미지는 패스워드 없이 누구나 포트로 접근이 가능하여 보안상 허점이 존재합니다.
- **반영 사항**: 컨테이너 실행 `command`를 덮어씌워 `--requirepass ${REDIS_PASSWORD}` 속성으로 비밀번호 보호를 활성화했습니다.

## 4. 실행 및 관리 방법

터미널 계열 프로그램에서 다음 명령어들을 활용합니다. (사전에 `.env` 파일과 `docker-compose.yml`이 같은 경로에 있어야 합니다.)

- **전체 실행**: `docker-compose up -d` (백그라운드 실행)
- **전체 종료 및 컨테이너 삭제**: `docker-compose down`
- **로그 모니터링**: `docker-compose logs -f`
